# Legends Never Die PRD

## 1. 产品定位

### 1.1 游戏类型

`Legends Never Die` 是一款 `Roguelike` 轻度成长向的横版自动跑酷射击游戏。

玩家控制主角在持续前进的横版场景中生存，通过：

- 规避障碍
- 击倒敌人
- 累积跑酷距离
- 升级并构筑成长流派

尽可能存活更久并获得更高分数。

### 1.2 游戏目标

- 核心目标：在不断提速和不断增强的敌人与障碍压力下尽可能长时间存活。
- 成长目标：通过击倒敌人获取经验，升级后从 3 张升级卡片中选择 1 张，逐步形成自己的构筑。
- 分数目标：跑得越久、击倒敌人越多、等级越高，最终得分越高。

### 1.3 目标体验

- 上手简单：只需跳跃、射击、导弹三个核心操作。
- 节奏紧凑：自动前进，玩家始终处于决策状态。
- 成长清晰：升级即停顿选卡，形成即时反馈。
- 构筑多样：武器、血量、护甲、飞行等升级可组合出不同生存路线。

## 2. 技术栈

### 2.1 桌面版

- 语言：`Java`
- UI/渲染：`Java Swing`
- 图形资源：位图精灵、背景图、UI 图标
- 音频：本地音效与背景音乐播放

### 2.2 Web 版

- 游戏框架：`Phaser 3`
- 语言：`TypeScript`
- 渲染：`Phaser 3 WebGL / Canvas`
- 本地开发与构建：`Vite`

### 2.3 设计原则

- 双端共享同一套玩法规格与数值逻辑
- 允许表现层不同，但核心机制保持一致
- Web 版优先使用 Phaser 3 管理场景、输入、动画、物理与音频
- 以跑酷、生存、成长构筑为主，不引入复杂剧情或重 UI 系统

## 3. 核心操作

### 3.1 桌面端操作

| 操作 | 按键 |
| --- | --- |
| 开始游戏 | `Enter` / `Space` |
| 跳跃 | `Space` |
| 射击 | `K` |
| 发射导弹 | `L` |
| 暂停 / 继续 | `P` / `ESC` |
| 选卡 1 / 2 / 3 | `1` / `2` / `3` |

### 3.2 移动端操作

Web 版触屏按钮：

- 跳跃
- 射击
- 导弹

升级选卡阶段采用点击卡片选择。

## 4. 核心玩法循环

## 4.1 单局循环

1. 玩家进入游戏后，角色自动向前奔跑。
2. 场景持续生成障碍、普通敌人、精英敌人、Boss。
3. 玩家通过跳跃、射击、导弹规避威胁并清理敌人。
4. 敌人被击倒后掉落经验，玩家累计经验升级。
5. 每次升级暂停战斗，弹出 3 张升级卡片，玩家选择 1 张。
6. 难度随时间和距离持续提升。
7. 当生命值归零时，游戏结束并结算得分。

### 4.2 Roguelike 核心

- 每局内成长随机生成
- 升级时从随机 3 选 1 的卡池中选择
- 同一局内构筑结果不同
- 局外不做永久数值成长，强调单局构筑体验

## 6. 得分与结算机制

### 6.1 得分公式

建议当前版本采用：

`最终得分 = 距离分 + 击杀分 + 等级分 + Boss 击杀分`

细化如下：

- 距离分：`距离(m) x 10`
- 普通敌人击杀分：`每个 20 分`
- 精英敌人击杀分：`每个 80 分`
- Boss 击杀分：`每个 200 分`
- 等级分：`每级 100 分`

### 6.2 结算展示

结算界面展示：

- 最终得分
- 跑酷距离
- 击杀总数
- Boss 击杀数
- 最终等级
- 本局构筑摘要

## 7. 敌人与障碍设计

### 7.1 障碍物

定位：纯地形压力单位。

| 项目 | 数值 |
| --- | --- |
| 生命值 | 不可摧毁 |
| 伤害 | `1` |
| 生成位置 | 地面 |
| 行为 | 随地图向左移动 |
| 被碰撞反馈 | 游戏短暂停顿 `1.0s` |

机制说明：

- 障碍物不能被子弹和导弹摧毁。
- 玩家主要通过跳跃或飞行规避。

### 7.2 普通敌人

定位：基础经验来源与输出练习目标。

| 项目 | 数值 |
| --- | --- |
| 生命值 | `1` |
| 接触伤害 | `1` |
| 经验值 | `5` |
| 分数 | `20` |
| 生成概率权重 | 高 |

### 7.3 精英敌人

定位：中期高价值目标。

| 项目 | 数值 |
| --- | --- |
| 生命值 | `3` |
| 接触伤害 | `1` |
| 经验值 | `15` |
| 分数 | `80` |
| 特性 | 速度略快，体型更大或有护盾表现 |

### 7.4 Boss

定位：阶段性压制单位。

| 项目 | 数值 |
| --- | --- |
| 生命值 | `8` |
| 接触伤害 | `1` |
| 导弹伤害 | `1` |
| 经验值 | `30` |
| 分数 | `200` |
| 首次出现距离 | `1200m` |
| 后续出现间隔 | 每 `1200m` 或 `90s` |
| 发射间隔 | `1.2s` |

机制说明：

- Boss 出现时应有明显音效和 UI 提示。
- Boss 导弹飞行速度高于普通弹体。
- Boss 被击倒后应给予明确的爆炸、停顿和奖励反馈。

## 8. 武器与战斗机制

### 8.1 普通子弹

| 项目 | 数值 |
| --- | --- |
| 基础伤害 | `1` |
| 飞行速度 | `8` |
| 穿透 | 否 |
| 初始射击数量 | `1` |

### 8.2 火力等级

火力等级决定单次射击发射的子弹数量。

| 火力等级 | 单次发射子弹数 |
| --- | --- |
| Lv1 | `1` |
| Lv2 | `2` |
| Lv3 | `3` |
| Lv4 | `4` |
| Lv5 | `5` |

说明：

- 超过 `Lv5` 后不再继续增加子弹数。
- 后续火力升级可转为伤害加成或射速加成。

### 8.3 导弹

| 项目 | 数值 |
| --- | --- |
| 基础伤害 | `2` |
| 飞行速度 | `6` |
| 穿透 | 是 |
| 默认持有 | `0` |
| 每次获得数量 | `+3` 或 `+5` |

机制说明：

- 导弹是稀缺资源型强力攻击。
- 适合清理精英敌人、Boss 或密集目标。

### 8.4 受击与护甲

- 玩家受击时，若有护甲，则先扣除护甲。
- 护甲不提供额外无敌，只用于抵消伤害。
- 玩家受击后统一进入 `0.30s` 无敌状态。
- 连续碰撞不会在同一瞬间叠加多段伤害。

## 9. 经验、升级与选卡机制

### 9.1 升级经验公式

升级需求采用递增机制：

- Lv1 -> Lv2：`10 XP`
- Lv2 -> Lv3：`20 XP`
- Lv3 -> Lv4：`30 XP`
- Lv4 -> Lv5：`40 XP`
- 之后每级继续 `+10 XP`

通式：

`下一级所需经验 = 当前等级 x 10`

### 9.2 升级流程

玩家经验达到阈值时：

1. 游戏暂停。
2. 弹出 3 张升级卡片。
3. 玩家选择 1 张。
4. 选中后立即生效。
5. 关闭选卡界面，游戏继续。

### 9.3 选卡规则

- 每次升级固定展示 `3` 张卡。
- 3 张卡不能完全重复。
- 已达到上限的升级不再进入候选池。
- 同类型成长允许多次抽到，直到达到上限。
- 稀有卡在前期出现概率较低，中后期提升。

### 9.4 卡片稀有度

| 稀有度 | 概率 |
| --- | --- |
| 普通 | `60%` |
| 稀有 | `30%` |
| 史诗 | `10%` |

### 9.5 升级卡池设计

#### A. 武器类

| 卡片名 | 效果 | 上限 |
| --- | --- | --- |
| 火力升级 | 火力等级 `+1` | `Lv5` |
| 射速升级 | 普通射击冷却 `-0.02s` | 最低 `0.10s` |
| 子弹强化 | 普通子弹伤害 `+1` | `+4` |
| 导弹补给 | 导弹 `+5` | 无 |
| 爆裂弹头 | 导弹爆炸半径伤害开启 | `1` |

#### B. 生存类

| 卡片名 | 效果 | 上限 |
| --- | --- | --- |
| 生命提升 | 当前生命 `+1`，最大生命 `+1` | `+5` |
| 护甲装置 | 护甲 `+2` | `10` |
| 紧急修复 | 恢复 `2` 点生命 | 不超过最大生命 |
| 韧性提升 | 受击无敌时间 `+0.1s` | `0.8s` |

#### C. 机动类

| 卡片名 | 效果 | 上限 |
| --- | --- | --- |
| 轻量化 | 跳跃高度提升 `15%` | `3` 次 |
| 飞行模块 | 获得 `10s` 飞行 | 无 |
| 飞行续航 | 飞行持续时间 `+3s` | `19s` |
| 起跳喷射 | 跳跃启动更快，起跳速度 `+10%` | `3` 次 |

#### D. 功能类

| 卡片名 | 效果 | 上限 |
| --- | --- | --- |
| 经验增幅 | 击杀经验获取 `+20%` | `3` 次 |

## 10. 主要机制设计

### 10.1 生成机制

场景单位按时间与距离混合驱动生成。

规则建议：

- 普通单位生成间隔：`0.8s - 1.8s`
- 最小安全间距：`650px`
- 前 `300m` 只生成障碍和普通敌人
- `300m - 1200m` 引入精英敌人
- `1200m` 后开始出现 Boss

### 10.2 难度曲线

难度提升来自四部分：

- 跑酷速度提升
- 生成频率提高
- 精英敌人与 Boss 介入
- 升级需求递增，迫使玩家更重视构筑选择

### 10.3 暂停规则

以下情况进入暂停：

- 玩家主动暂停
- 升级选卡
- 游戏结束

升级暂停属于强制暂停，不允许跳过。

### 10.4 碰撞规则

- 玩家与障碍、敌人本体、Boss 导弹发生矩形碰撞时判定受伤。
- 玩家子弹命中普通敌人、精英敌人、Boss 时造成伤害。
- 导弹可穿透多个敌人。
- 障碍不受伤害。

### 10.5 飞行规则

- 飞行期间角色处于固定高空轨道。
- 飞行时不需要跳跃输入。
- 飞行状态结束后角色回到地面默认跑酷状态。
- 若结束点正好位于障碍上方，系统应将角色平滑回落至最近安全落点。

## 11. 单局目标与里程碑

### 11.1 前期

`0m - 800m`

- 目标：熟悉躲避和基础射击
- 推荐卡方向：火力升级、生命提升、护甲装置

### 11.2 中期

`800m - 2400m`

- 目标：稳定处理精英敌人与第一轮 Boss
- 推荐卡方向：射速升级、导弹补给、飞行模块

### 11.3 后期

`2400m+`

- 目标：围绕成型构筑极限生存
- 推荐卡方向：伤害叠加、经验增幅、飞行续航、赏金模块

## 12. 当前版本验收标准

1. 玩家可在 30 秒内理解全部操作。
2. 游戏单局能完整体验跑酷、战斗、升级选卡、结算流程。
3. 升级时必须弹出 3 张卡片并可选择其一。
4. 至少支持武器、生存、机动三类升级路线。
5. 距离越长、等级越高、击杀越多，得分越高。
6. 玩家能通过构筑差异明显感受到不同局内体验。
7. Web 版和 Java 版在核心机制上保持一致。
